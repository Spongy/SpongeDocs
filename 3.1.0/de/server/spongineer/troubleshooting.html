Fehlerbehebung¶

Du bist wahrscheinlich hier gelandet, weil etwas mit deinem Sponge Server schief lief. Lass uns ermitteln was das war und was du dagegen tun kannst.

Mögliche Fehlerquellen

  • Java ist nicht auf deinem Computer installiert
  • Netzwerkverbindungsfehler (oder DDos-Attacke)
  • Nicht genügend freier Arbeitsspeicher verfügbar
  • Fehlerhafte Konfigurationsdatei (z.B. durch falsches Editieren)
  • Ein Plugin (oder eine Mod) funktioniert nicht
  • Instabiles Betriebssystem (z. B. durch einen Computervirus)
  • Beschädigte Daten
  • Problem zwischen Tastatur und Stuhl
  • Es gibt einen Bug in Sponge
  • Da ist etwas falsch mit dem Universum.

Java ist nicht auf deinem Computer installiert¶

Lösung: Installiere Java. Lese Java installieren für weitere Informationen.

Netzwerkverbindungsfehler (oder DDos-Attacke)¶

Symptome: Die Netzwerkverbindung ist sehr langsam oder nicht verfügbar.

Lösung: Prüfe die Verbindung deines Modems oder Routers. Prüfe außerdem, ob dein Browser ebenfalls Probleme mit dem Internetzugriff hat. Zum Test kann ein Dienst wie z.B. speedtest.net genutzt werden, um die Verbindungsgeschwindigkeit zu ermitteln. Andere Services, die auf deinem Computer oder im lokalen Netzwerk laufen könnten der Auslöser der Probleme sein. Stelle sicher, dass Port Forwarding in deinem Router richtig konfiguriert ist. Eine DDoS Attacke, so unwahrscheinlich diese auch ist, wird deine Verbindung zusammenbrechen lassen und du solltest einen Internet Provider (ISP) kontaktieren, falls dies der Fall ist.

Nicht genügend freier Arbeitsspeicher verfügbar¶

Symptome: Der Server stürzt ab, oft begleitet von “Out of Memory”-Meldungen.

Lösung: Erweitere die maximale permanente Speichergröße mit dem Startargument -XX:MaxPermSize=128. Erweitere deinen Server-Heap-Speicher (falls möglich) z.B. mit dem Startargument -Xms1024M (1GB Startspeicher) oder -Xmx2048M (2GB maximal). Beobachte den freien Speicher deines Computers und suche nach ungenutztem Speicher in anderen Prozessen. Du musst vielleicht eingefrorene Java-Prozesse beenden oder den Computer neu starten. Memory Leaks werden manchmal durch Bugs in Plugins ausgelöst. Es ist zeitaufwendig diese Plugins durch Ausprobieren aufzuspüren.

Hast du immer noch ein Problem?: Wenn du immer noch, trotz den oben genannten Hilfestellungen Probleme hast und nicht die “Heap-Size” erhöhen kannst, öffne deinen Task Manager und überprüfe, ob du genügend RAM frei hast. Wenn du nicht genug frei hast, ist die einzige Möglichkeit, mehr RAM nachzurüsten. Wenn du aber genug RAM frei ist, kann es sein das du Java 32-Bit benutzt. Wenn du Java 32-Bit benutzt empfehlen wir ein Upgrade auf Java 64-Bit (sofern dein Betriebssystem auch 64-Bit ist).

Fehlerhafte Konfigurationsdatei (z.B. durch falsches Editieren)¶

Symptom: Ein (oder mehrere) Plugins starten nicht oder verhalten sich unerwartet. Die Server-Logs werden zeigen beim Serverstart Meldungen über unlesbare Dateien an. Der Server stürzt möglicherweise ab und Spieldaten können beschädigt sein.

Lösung: Stoppe den Server und überprüfe deine editierten Dateien. Lade Backupdateien der fehlerhaften Daten. Du musst möglicherweise eine Konfigurationsdatei komplett löschen, um sie beim Serverstart neu generieren zu lassen.

Ein Plugin (oder eine Mod) funktioniert nicht¶

Symptom: Es gibt die verschiedensten Kennzeichen dafür - das hängt davon ab, was deine Plugins machen, aber auch von vielen weiteren unabsehbaren Faktoren. Gewöhnlich stürzt der Server begleitet von einer Folge von Fehlermeldungen in den Server-Log-Dateien ab.

Lösung: Stoppe den Server und prüfe, dass keine Spieldaten beschädigt wurden. Stelle sicher, dass keine kürzlich editierte Konfigurationsdatei fehlerhaft ist (siehe oben). Entferne möglicherweise störende Plugins und füge sie nacheinander wieder hinzu. Starte den Server jedes Mal neu, nachdem zu ein Plugin hinzugefügt hast. Das Problem kann auch von einem veralteten Plugin stammen. Prüfe deshalb, ob Updates für deine Plugins verfügbar sind. Ein Plugin-Konflikt kann auch die Ursache des Problems sein. Dieser entsteht bei untereinander inkompatiblen Plugins.

Instabiles Betriebssystem (z. B. durch einen Computervirus)¶

Symptom: Der Server stürzt immer ab oder es kommt zu einem “Timeout”, auch andere Bereiche deines Betriebssystems haben Probleme.

Lösung: Stoppe alles. Überprüfe gründlich dein System und Speichergeräte auf Malware und Viren. Geeignete Programme dafür sind z.B. AdwCleaner, Junkware Removal Tool, MalwareBytes und die meisten Antivirusprogramme. Nach einem ordnungsgemäßen, sauberen Start deines Systems solltest du deine Server-Dateien auf Beschädigungen überprüfen. Prüfe auch, ob die Hardware deines PCs beschädigt wurde, wenn weiterhin Probleme bestehen - z. B. ein defektes Netzteil.

Beschädigte Daten¶

Symptom: Die Welt-Dateien laden nicht oder verursachen einen Server-Absturz, wenn Spieler einen bestimmten Chunk betreten; Datenbankbeschädigung.

Lösung: Lade Backups von den beschädigten Daten. Es existiert Software für das Reparieren von beschädigten Welten. Fehlende Regionen können so möglicherweise wiederhergestellt werden. Untersuche den Grund für die defekten Daten - war es ein fehlerhaftes Plugin, ein Datenbanktreiber, ein Stromausfall oder etwas anderes? Stelle immer sicher, dass du regelmäßig Backups von wichtigen Daten auf externen, sicheren Speichermedien erstellst.

Problem zwischen Tastatur und Stuhl¶

Symptom: Gestern funktionierte alles problemlos. Es wurde merkwürdig nach dem ich XYZ gemacht habe...

Lösung: Diese Dokumentation kann niemals alle Dinge umfassen, die Leute möglicherweise machen und damit Software in unvorhersehbarer Weise zum Abstürzen bringen. Es lohnt sich immer, lange und genau über kürzliche Ereignisse nachzudenken, die den reibungslosen Ablauf deines Servers beeinträchtigt haben könnten. Eine Speicherkarte könnte beim Entstauben deines PCs verloren gegangen sein, oder eine Verknüpfung ist defekt...

Es gibt einen Bug in Sponge¶

Symptom: Nichts des oben genannten trifft zu und es funktioniert immer noch nicht, wie es sollte.

Lösung: Zeit die großen Geschütze aufzufahren. Erstellen einen Bericht auf dem SpongeForge oder SpongeVanilla Issue Tracker. Beachte das du die Details deines Setups (Serverversion, Forge Version etc.) mit angibst und am Besten eine Log-Datei mit anhängst.

Da ist etwas falsch mit dem Universum.¶

Hierbei können wir dir nicht helfen. Du bist auf dich allein gestellt.
