Dépannage¶

Vous êtes probablement ici parce que quelque chose s’est mal passé avec votre serveur Sponge. Nous allons voir si nous pouvons comprendre ce que c’était et ce qu’il faut faire à ce sujet.

Source potentielle de problème

  • Java n’est PAS installé sur votre ordinateur
  • Connexion au réseau échouée (ou attaque DDoS)
  • Mémoire libre insuffisante
  • Fichier de configuration mal formé (ex. Faute de frappe)
  • Un plugin (ou un mod) a mal fonctionné
  • Système d’exploitation instable (ex. Infection)
  • Données corrompues
  • Problème entre la chaise et votre clavier
  • Sponge: “J’ai contracté un bug”
  • Il y a quelque chose qui ne tourne pas rond.

Java n’est PAS installé sur votre ordinateur¶

Solution: téléchargez Java. Visitez la page Installation de Java pour plus d’informations.

Connexion au réseau échouée (ou attaque DDoS)¶

Symptômes: Connexion au réseau très lente, trop fluctuante ou absente.

Solutions: Vérifiez votre connexion au modem ou au routeur. Vérifiez si votre navigateur à un problème similaire. Vous pouvez utiliser un service gratuit comme speedtest.net pour vérifier votre vitesse de connexion. Les autres services tournant sur votre machine ou sur votre réseau local peuvent en être la cause. Vérifiez que vous avez bien activé la redirection de port sur votre routeur. Une attaque par dénis de service DDOS peut complètement surcharger vote connexion. Si vous pensez que vous êtes victime d’une telle attaque, contactez votre FAI.

Mémoire libre insuffisante¶

Symptômes: Les incidents du serveur sont souvent accompagnés du message “Out of memory”

Solutions: La taille de la mémoire de Perm maximale avec l’argument de démarrage `` -XX: MaxPermSize = 128``. Développez votre mémoire des serveur (si possible) avec des arguments de démarrage par exemple. `` -Xms1024M`` (1 Go de mémoire de départ) et `` -Xmx2048M`` (maximum de 2 Go). Surveillez votre mémoire libre sur votre l’ordinateur pour voir si il y a une certaine enfermé dans d’autres processus. Vous devrez peut-être arrêter les processus en java, ou de redémarrer votre machine. Les fuites de mémoire se produisent parfois avec des bugs de plugins, ce qui peut prendre du temps à s’isoler.

Encore un problème?: Si vous avez toujours un problème et ne pouvez pas augmenter la mémoire, ouvrez le gestionnaire de tâches pour vérifier que vous utilisez bien toutes la mémoire de votre système. SI vous l’utilisez bien, la seule solution est d’ajouter plus de RAM sur votre système. Si il reste de la mémoire disponible, cela veut dire que vous utilisez java 32 bits. Nous vous recommandons donc d’installer java 64 bits, ainsi qu’un système d’exploitation 64 bits

Fichier de configuration mal formé (ex. Faute de frappe)¶

Symptôme: Un (ou plusieurs) plugins refusent de charger, ou se comportent de manière inattendue. Le journal du serveur contiendra des messages à propos des fichiers illisibles au démarrage. Le serveur peut crasher, et les données peuvent être corrompues.

Solution: Arrêtez le serveur, et vérifiez vos fichiers édités. Chargez les backups de tous les fichiers corrompus. Vous aurez peut-être besoin de supprimer un fichier de configuration et l’autoriser à régénérer un fichier de démarrage du serveur.

Un plugin (ou un mod) a mal fonctionné¶

Symptôme: Ça pourrait être n’importe quoi - quoi que vos plugins fassent, en plus l’agent chimique X. Généralement votre serveur crash avec pleins de messages d’erreur dans le journal du serveur

Solution : Arrêtez le serveur, et vérifiez si rien n’est corrompu. Soyez sûr que le fichier de configuration soit correctement modifié (ci-dessus). Enlevez les plugins suspects, et ajoutez les uns par uns, et redémarrer le serveur autant de fois que vous avez de plugins. Le problème peut être du au fait que votre plugin n’est pas a jour - vérifiez régulièrement s’il y a des mises à jours. Les conflits entre plugins peuvent aussi en être la cause, vous avez peut être 2 plugins incompatibles entre eux.

Système d’exploitation instable (ex. Infection)¶

Symptôme: Le serveur n’arrête pas de crasher ou de se désynchroniser, et d’autres parties de votre système d’exploitation ont aussi des problèmes.

Solution: Arrêtez tout. Vérifiez soigneusement votre système et votre matériel de stockage si il y a des logiciels malveillants et des virus. Il y a de bons outils pour ça comme AdwCleaner, Junkware Removal Tool, MalwareBytes, et la plupart des antivirus. Vérifiez les fichiers du serveur pour détecter toute trace de corruption après un redémarrage complet de votre système. Examinez aussi votre matériel informatique pour détecter des dommages si les problèmes persistent - ex. un transformateur défectueux

Données corrompues¶

Symptôme: Les fichiers du monde ne réussissent pas à charger ou le serveur crash dès qu’un joueur entre dans certains chunks. Base de données corrompues.

Solution: Chargez les fichiers de sauvegarde de données corrompues. Un logiciel pour réparer les mondes endommagés est disponible, et les régions manquantes peuvent être régénérées. Enquêtez la cause de corruption - était-ce un plugin mal formé, un disque dur défaillant ou une panne d’électricité ou autre chose? Soyez sûr de faire des sauvegardes de données importantes couramment dans un espace de stockage sûr.

Problème entre la chaise et votre clavier¶

Symptôme: Tout allait bien hier. Aujourd’hui je sais pas ce qui c’est passé, mais ça lui a pas plu...

Solution: Sponge Docs n’est pas assez grand pour englober les choses que les gens peuvent faire qui vont entraîner un logiciel à l’échec de manière imprévisible. Il est toujours utile de réfléchir longuement et sérieusement à ce que vous avez fait récemment qui aurait pu affecter le bon fonctionnement de votre serveur. Une carte mémoire peut être lâche après dépoussiérage, un raccourci peut être cassé ...

Sponge: “J’ai contracté un bug”¶

Symptôme: Aucune de ces réponses ne s’applique, et ça ne marche toujours pas comme ça le devrait.

Solution: Il est temps de sortir l’artillerie lourde. Déposez un rapport sur le traqueur d’issues de SpongeForge ou de SpongeVanilla, rappelez-vous d’inclure tous les détails concernant la version de Forge et de Sponge que vous utilisez, et un lien menant vers le log concerné de votre serveur.

Il y a quelque chose qui ne tourne pas rond.¶

Nous ne pouvons pas vous aider avec celui-ci. Trouvez une solution par vous-même.
