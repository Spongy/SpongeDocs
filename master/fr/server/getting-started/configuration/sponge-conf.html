global.conf¶

Configuration globale¶

Le fichier global.file contient les paramètres de configuration globale de Sponge. Ce fichier est créé dans le dossier config/sponge du répertoire de votre serveur. Beaucoup de ces paramètres peuvent être remplacés par une configuration par monde ou par dimension en utilisant le fichier de configuration du dossier config/worlds.

Below is a complete global.conf file with all possible nodes that may be present on a server. Note that certain sections will not be present immediately, and will be added to the file when the server encounters them. This config was generated using SpongeForge build 1185, SpongeAPI version 4.0.2:

# 1.0
#
# # If you need help with the configuration or have any questions related
# # to Sponge, join us at the IRC or drop by our forums and leave a post.
#
# # IRC: #sponge @ irc.esper.net ( http://webchat.esper.net/?channel=sponge )
# # Forums: https://forums.spongepowered.org/
#

sponge {
    block-tracking {
        # Add block ids you wish to blacklist for player block placement tracking.
        block-blacklist=[]
        # If enabled, adds player tracking support for block positions. Note: This should only be disabled if you do not care who caused a block to change.
        enabled=true
    }
    bungeecord {
        # If enabled, allows BungeeCord to forward IP address, UUID, and Game Profile to this server
        ip-forwarding=false
    }
    commands {
        # A mapping from unqualified command alias to plugin id of the plugin that should handle a certain command
        aliases {}
    }
    debug {
        # Dump chunks in the event of a deadlock
        dump-chunks-on-deadlock=false
        # Dump the heap in the event of a deadlock
        dump-heap-on-deadlock=false
        # Dump the server thread on deadlock warning
        dump-threads-on-warn=false
        # Enable Java's thread contention monitoring for thread dumps
        thread-contention-monitoring=false
    }
    entity {
        # Number of colliding entities in one spot before logging a warning. Set to 0 to disable
        collision-warn-size=200
        # Number of entities in one dimension before logging a warning. Set to 0 to disable
        count-warn-size=0
        # Number of ticks before a painting is respawned on clients when their art is changed
        entity-painting-respawn-delay=2
        # Number of ticks before the fake player entry of a human is removed from the tab list (range of 0 to 100 ticks).
        human-player-list-remove-delay=10
        # Controls the time in ticks for when an item despawns.
        item-despawn-rate=6000
        # Max size of an entity's bounding box before removing it. Set to 0 to disable
        max-bounding-box-size=1000
        # Square of the max speed of an entity before removing it. Set to 0 to disable
        max-speed=100
    }
    entity-activation-range {
        ambient-activation-range=32
        aquatic-activation-range=32
        creature-activation-range=32
        minecraft {
            ambient {
                bat=true
            }
            aquatic {
                squid=true
            }
            creature {
                chicken=true
                cow=true
                entityhorse=true
                mushroomcow=true
                ocelot=true
                pig=true
                rabbit=true
                sheep=true
                wolf=true
            }
            enabled=true
            misc {
                armorstand=true
                arrow=true
                boat=true
                fallingsand=true
                item=true
                itemframe=true
                leashknot=true
                minecartchest=true
                minecartfurnace=true
                minecarthopper=true
                minecartrideable=true
                minecarttnt=true
                villager=true
                villagergolem=true
                xporb=true
            }
            monster {
                blaze=true
                cavespider=true
                creeper=true
                enderman=true
                endermite=true
                ghast=true
                giant=true
                guardian=true
                lavaslime=true
                pigzombie=true
                silverfish=true
                skeleton=true
                slime=true
                snowman=true
                spider=true
                witch=true
                zombie=true
            }
        }
        misc-activation-range=16
        monster-activation-range=32
    }
    exploits {
        prevent-creative-itemstack-name-exploit=true
        prevent-sign-command-exploit=true
    }
    general {
        # Forces Chunk Loading on provide requests (speedup for mods that don't check if a chunk is loaded)
        chunk-load-override=false
        # Disable warning messages to server admins
        disable-warnings=false
    }
    ip-sets {}
    logging {
        # Log when blocks are broken
        block-break=false
        # Log when blocks are modified
        block-modify=false
        # Log when blocks are placed
        block-place=false
        # Log when blocks are populated in a chunk
        block-populate=false
        # Log when blocks are placed by players and tracked
        block-tracking=false
        # Log when chunks are loaded
        chunk-load=false
        # Log when chunks are unloaded
        chunk-unload=false
        # Whether to log entity collision/count checks
        entity-collision-checks=false
        # Log when living entities are destroyed
        entity-death=false
        # Log when living entities are despawned
        entity-despawn=false
        # Log when living entities are spawned
        entity-spawn=false
        # Whether to log entity removals due to speed
        entity-speed-removal=false
        # Log when server receives exploited packet with itemstack name exceeding string limit.
        exploit-itemstack-name-overflow=false
        # Log when player attempts to respawn invisible to surrounding players.
        exploit-respawn-invisibility=false
        # Log when server receives exploited packet to update a sign containing commands from player with no permission.
        exploit-sign-command-updates=false
        # Add stack traces to dev logging
        log-stacktraces=false
    }
    modules {
        bungeecord=false
        entity-activation-range=true
        exploits=true
        timings=true
    }
    # Configuration options related to the Sql service, including connection aliases etc
    sql {
        # Aliases for SQL connections, in the format jdbc:protocol://[username[:password]@]host/database
        aliases {}
    }
    timings {
        enabled=true
        hidden-config-entries=[
            "sponge.sql"
        ]
        history-interval=300
        history-length=3600
        server-name-privacy=false
        verbose=false
    }
    world {
        # Lava behaves like vanilla water when source block is removed
        flowing-lava-decay=false
        # Enable if you want the world to generate spawn the moment its loaded.
        generate-spawn-on-load=true
        # Vanilla water source behavior - is infinite
        infinite-water-source=false
        # Enable if this world's spawn should remain loaded with no players.
        keep-spawn-loaded=true
        # Enable if this world should load on startup.
        load-on-startup=true
        # Enable if this world allows PVP combat.
        pvp-enabled=true
        # Enable if this world should be allowed to load.
        world-enabled=true
    }
}

Propriétés globales de Sponge¶

Propriété

 Type

Défaut

 Description

Suivi des Blocs

      
block-blacklist

string (chaîne de caractères)

 null

Ajoute l’ID d’un block que vous souhaitez ajouter à la liste noire du suivi des blocks placés par un joueur.
enabled boolean true

Ajoute le suivi du joueur pour les positions des blocs.
Bungeecord      
ip-forwarding boolean false

Autorise bungeecord à transmettre adresse ip, UUID et profil de jeu au serveur.

Commandes

      

alias

string (chaîne de caractères)

 null

L’alias résoudra les conflits lorsque plusieurs plugins utilisent une même commande. La syntaxe correcte est <unqualified command>: <plugin name> Exemple:

aliases = {
    title=myPlugin
}

Options de Debug

      
dump-chunks-on-deadlock boolean false

Supprime les chunks dans le cas d’un blocage (deadlock).
dump-heap-on-deadlock boolean false

Supprime le surplus dans le cas d’un blocage (deadlock).
dump-threads-on-warn boolean false

Supprime le thread du serveur lors d’un avertissement de blocage.
thread-contention-monitoring boolean false

Active la surveillance de Java sur les thread supprimés.

Options des Entités

      
collision-warn-size integer 200

Nombre d’entités au même endroit avant de vous avertir. La valeur 0 désactive cette option.
count-warn-size integer 0

Nombre d’entités autorisées à se trouver dans une même dimension avant de vous avertir. La valeur 0 désactive cette option.
entity-painting-respawn-delay integer 2

Nombre de ticks avant qu’une peinture ne réapparaisse chez le client quand leur contenu est changé.
human-player-list-remove-delay integer 10

Nombre de ticks avant que l’entrée du faux joueur d’un humain ne soit supprimé de la tab list. La plage autorisée est 0 - 100.
item-despawn-rate integer 6000

Le temps en ticks avant qu’un item despawn.
max-bounding-box-size integer 1000

Taille maximale de la zone englobant l’entité avant de la supprimer. La valeur 0 désactive cette option.
max-speed integer 100

Carré de la vitesse maximale d’une entité avant qu’elle ne soit supprimée, La valeur 0 désactive cette option

Rayon d’activation des entités

    

Notez que les entités venant des Mods sont automatiquement ajoutées à cette liste
ambient-activation-range integer 32

Rayon dans lequel les entités ambiantes deviennent actives.
aquatic-activation-range integer 32

Rayon dans lequel les entités aquatiques deviennent actives.
creature-activation-range integer 32

Rayon dans lequel les créatures deviennent actives.

Créatures de Minecraft

      
ambient (bat) boolean true

Utilise le rayon d’activation sur les bats.
aquatic (squid) boolean true

Utilise le rayon d’activation sur les squid.
creature <creature> boolean true

Utilise le rayon d’activation sur <creature>. Valeurs autorisées: chicken, cow, entityhorse, mushroomcow, ocelot, pig, rabbit, sheep, wolf.
enabled boolean true

Active le rayon d’activation sur les entités Minecraft.

Entités Diverses

      
misc <entity> boolean true

Utiliser le rayon d’activation sur <entity>. Valeurs autorisées: armorstand, arrow, boat, fallingsand, item, itemframe, leashknot, minecartchest, minecartfurnace, minecarthopper, minecartrideable, minecarttnt, villager, villagergolem, xporb.

Monstres de Minecraft

      
monster <monster> boolean true

Utiliser le rayon d’activation sur <monster>. Valeurs autorisées: blaze, cavespider, creeper, enderman, endermite, ghast, giant, guardian, lavaslime, pigzombie, silverfish, skeleton, slime, snowman, spider, witch, zombie.
misc-activation-range integer 16

Rayon dans lequel les entités diverses deviennent actives.
monster-activation-range integer 32

Rayon dans lequel les monstres deviennent actifs.
Exploits      
prevent-creative-itemstack-name-exploit boolean true

Empêche qu’une faille dans laquelle le client envoie un packet portant le nom d’un itemstack, dépassant la limite de la chaîne de caractères.
prevent-sign-command-exploit boolean true

Empêche une faille dans laquelle le client envoie un packet pouvant à mettre à jour un panneau contenant des commandes provenant d’un joueur sans permission.

Paramètres Généraux

      
chunk-load-override boolean false

Force le chargement des chunks sur demande. Cela permet d’accélérer les mods qui ne check pas si un chunk est chargé.
disable-warnings boolean false

Désactive les messages d’avertissements aux Admins du serveur.
Ip Sets      
     

Options de Logging

      
block-break boolean false

Log lorsqu’un bloc est cassé.
block-modify boolean false

Log lorsque des blocs sont modifiés.
block-place boolean false

Log lorsque des blocs sont placés.
block-populate boolean false

Log lorsque les blocs remplissent un chunk.
block-tracking boolean false

Log lorsque les blocs sont placés par un joueur et traqués.
chunk-load boolean false

Log lorsque les chunks se chargent.
chunk-unload boolean false

Log lorsque les chunks se déchargent.
entity-collision-checks boolean false

Log les checks de collision/compte d’entités.
entity-death boolean false

Log lorsqu’une entité a été détruite.
entity-despawn boolean false

Log lorsqu’une entité a despawn.
entity-spawn boolean false

Log lorsqu’une entité a spawn.
entity-speed-removal boolean false

Log lorsqu’une entité a été supprimé à cause de sa vitesse.
exploit-itemstack-name-overflow boolean false

Log lorsque le serveur reçoit un packet d’exploitation avec le nom d’un itemstack qui dépasse la limite de texte.
exploit-respawn-invisibility boolean false

Log lorsqu’un joueur tente de respawn invisible pour les joueurs environnants.
exploit-sign-command-updates boolean false

Log lorsque le serveur reçoit un packet d’exploitation contenant la commande d’un joueur qui n’a pas la permission de l’utiliser.
log-stacktraces boolean false

Ajoutez un tas de traceurs pour le développement.
Modules      
bungeecord boolean false

Active le support de bungeecord.
entity-activation-range boolean true

Activer les options du rayon d’action des entités.
exploits boolean true

Active le module de prévention d’exploit.
timings boolean true

Active les paramètres de synchronisation.
SQL      

alias

string (chaîne de caractères)

 null

Alias pour les connections SQL. Cela se fait dans le format jdbc:protocol://[username[:password]@]host/database
Timings      
enabled boolean true

Si les timings sont activés.
hidden-config-entries

string (chaîne de caractères)

 sponge.sql

Les entrées cachées de la configuration.
history-interval integer 300

L’intervalle entre la génération de deux rapports de timings.
history-length integer 3600

Combien de temps, en ticks, l’historique des timings sera gardé pour le serveur.
server-name-privacy boolean false

Si vous souhaitez inclure des informations comme le nom du serveur, motd, online-mode, et server icon dans le rapport.
verbose boolean false

Si oui ou non les timings sont surveillés à un niveau verbal élevé.

Paramètres du Monde

      
flowing-lava-decay boolean false

La lave se comporte comme l’eau en vanilla quand la source est enlevée lorsque la valeur est true.
generate-spawn-on-load boolean true

Si le monde doit générer le spawn quand le monde est chargé.
infinite-water-source boolean false

False = Comportement vanilla des sources d’eau par défaut.
keep-spawn-loaded boolean true

Si le spawn doit rester chargé sans joueur.
load-on-startup boolean true

Si le monde doit être chargé durant le démarrage du serveur.
pvp-enabled boolean true

Si le PVP est autorisé.
world-enabled boolean true

Activé si le monde est autorisé à être chargé.
