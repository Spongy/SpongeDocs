Les concepts de base¶

L’API Economie de Sponge possède quelques éléments de base que les développeurs doivent connaître:

  • Le service d’Economie

  • La devise

  • Les comptes

  • Les transactions

Le service d’économie¶

Le ‘’Service d’Economie’’ est la base de l’API Economie. Il est utilisé pour interagir via l’API Economie, stocker les monnaies et fournir des méthodes pour la gestion des comptes. Les JavaDocs du ‘’Service d’Economie’’ peuvent être trouvés ‘ici <https://jd.spongepowered.org/index.html?org/spongepowered/api/service/economy/EconomyService.html>’ __.

Avertissement

Sponge ne fournit pas d’implémentation par défaut pour le ‘’Service d’Economie’’ . C’est aux plugins d’implémenter l’API Economie. Cela signifie que vous devez dépendre d’un autre plugin implémentant l’API ou alors l’implémenter vous même si vous voulez l’utiliser.

La devise¶

L’objet org.spongepowered.api.service.economy.Currency représente une sorte de Monnaie. ‘’Currency’’ stocke un nom d’affichage, display name (Au pluriel et au singulier), un symbole, la quantité de nombres significatifs, et si la monaie est la monnaie par défaut de l’économie. Si le plugin d’économie est fait pour, il peut supporter plusieurs monnaies.

Les comptes¶

Les comptes sont utilisés pour stocker les informations économiques par rapport à un joueur spécifique où un autre objet (ex: Une banque, une entreprise, une entité,...). Il y a deux types de comptes dans l’API Economie de Sponge:

Les comptes virtuels¶

Les comptes virtuels sont reliés à un identifiant qui est déclaré dans un String. Les comptes virtuels peuvent être reliés à presque tout, mais ils sont communément utilisés pour des choses comme les banques, ou des entités non-joueuses. Pour récupérer l’identifiant d’un compte virtuel, utilisez ‘’getIdentifier()’‘.

Les comptes uniques¶

Les comptes uniques sont relités à un UUID, habituellement un joueur. Pour récupérer l’UUID d’un compte unique, utilisez ‘’getUUID()’‘.

Les transactions¶

Les transactions représentent les changements de solde d’un compte. Il y a actuellement trois types de transactions:

  • Dépot: A lieu quand des fonds sont ajoutés à un compte

  • Retrait: A lieu quand un des fonds sont retirés d’un compte

  • Transfert: A lieu quand un compte échange des fonds avec un autre compte

Quand un transfert a lieu, l’event ‘’EconomyTransactionEvent’’ est déclenché. En utilisant cet event, vous pouvez récupérer le ‘’TransactionResult’‘. Le ‘’TransactionResult’’ stocke les données de la transaction qui a eu lieu, cela inclut:

  • Le(s) compte(s) impliqué(s)

  • La monnaie impliquée

  • La quantité d’argent impliquée

  • Le type de transaction

  • Le résultat de la transaction

Vous pouvez voir la liste des résultats de transaction possible ici: Sponge JavaDocs.
