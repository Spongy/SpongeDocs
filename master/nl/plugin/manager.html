De Plugin Manager gebruiken¶

De Plugin Manager is hetgeen waar jouw plugin heen wordt gestuurd nadat hij geladen wordt door de server bij het opstarten. De server laadt de plugin door het zoeken van de main klasse die geannoteerd is door de @Plugin annotatie welke algemene informatie bevat en stuurt vervolgens deze nieuwe instantie naar de manager. De manager houdt deze instantie dan bij in zijn eigen verzameling die je kan inkijken en gegevens uit kan halen met behulp van methodes die de manager aanbiedt, wat het dus heel gemakkelijk maakt om te interageren met andere ingelade plugins.

De PluginManager Klasse¶

Tip

See the Javadocs for PluginManager for full information about the class, as well as its methods and their usage.

Publieke methodes in de PluginManager worden gebruikt om informatie te verkrijgen over de huidige verzameling ingelade plugins en om de instanties zelf te verkrijgen. De plugins worden in een PluginContainer (besproken in het volgende deel) gestopt om een gemakkeiljke verzamelplaats van informatie te voorzien van de specifieke plugin. Je kan bijvoorbeeld de PluginManager gebruiken om te communiceren met een andere plugin door zijn instantie te nemen en de methodes die de plugin aanbiedt te gebruiken om compatibiliteit met deze plugin aan te bieden of zijn functies uit te breiden.

De Plugin Manager verkrijgen¶

Je kan op verschillende manieren een instantie van de servers PluginManager verkrijgen.

1. Dependency Injection¶

Tip

Zie de :doc:`injection`handleiding voor hulp bij het gebruik van dependency injection.

De PluginManager is een van de weinige API instanties die geïnjecteerd wordt in de main klasse wanneer het geladen wordt. Om een referentie naar de instantie te verkrijgen maak je simpelweg een nieuwe variabele aan waar de PluginManager instantie kan gestoken worden en annoteer je deze met @Inject.

import com.google.inject.Inject;
import org.spongepowered.api.plugin.PluginManager;

@Inject
private PluginManager pluginManager;

2. The Service Manager¶

Tip

Zie Werken met Services voor een volledige handleiding over de Service Manager.

De service manager houdt ook een instantie bij van de servers PluginManager. Gebruik simpelweg de methode ServiceManager.provide() met de PluginManager klasse (PluginManager.class) als parameter.

private PluginManager pluginManager = serviceManager.provide(PluginManager.class);

3. The Game Instance¶

Tip

See the Javadocs for Game for full information about the class, as well as its methods and their usage.

Een game instance kan eveneens een referentie naar de servers PluginManager voorzien.

private PluginManager pluginManager = game.getPluginManager();

Nu dat je een instantie hebt van de plugin manager, laten we het gebruiken.

4. Using the Sponge Class¶

The Sponge class works similarly to Game, with the exception that since Sponge contains static methods, it can be accessed anywhere throughout your plugin. You also do not need to store an instance of it, as you would need to do with Game.

import org.spongepowered.api.Sponge;

private PluginManager pluginManager = Sponge.getPluginManager();

De Plugin Manager gebruiken¶

De plugin manager voorziet enkele methodes om te werken met plugins.

Een heleboel methodes geven plugin containers terug, die we bespreken in het volgende deel. Plugin containers zij vrij vanzelfsprekend “containers” die de eigenlijke plugin bevatten.

Met de plugin manager wordt het mogelijk om alle plugins te verkrijgen die momenteel zijn geladen via de plugin manager:

import org.spongepowered.api.plugin.PluginContainer;

import java.util.List;

private List<PluginContainer> plugins = pluginManager.getPlugins();

Of, het is mogelijk om een instantie van de plugin container te verkrijgen zoals wordt aangetoond in het onderstaande voorbeeld:

private PluginContainer myOtherPlugin = pluginManager.getPlugin("myOtherPluginId").orNull();

De PluginContainer Klasse¶

Tip

See the Javadocs for PluginContainer for full information about the class, as well as its methods and their usage.

Wanneer je een plugin neemt van de PluginManager zal je al snel opmerken dat je niet meteen een instantie krijgt van de gevraagde plugin. In de plaats daarvan zal je een PluginContainer krijgen die informatie bevat over de plugin dat verkregen is via de @Plugin annotatie in de main klasse van de plugin en de ingelade instantie zelf.

De PluginContainer zal generische informatie hebben over de plugin die ingesteld is door de ontwikkelaar. Je kan deze informatie gebruiken in plaats van zelf informatie te hard-coden die je misschien weet over de plugin. Stel dat de ontwikkelaar van een plugin beslist om de naam van de plugin te veranderen. Idien er gebruik wordt gemaakt van voorgenoemde informatie zal deze informatie automatisch aangepast worden in jouw plugin zodat het blijft kloppen, zolang je gebruik hebt gemaakt van de PluginContainer.getName() methode om de naam te verkrijgen.

private PluginContainer myOtherPlugin = pluginManager.getPlugin("myOtherPluginId").orNull();
private MyOtherPlugin pluginInstance = (MyOtherPlugin) myOtherPlugin.getInstance();

Notitie

PluginContainer.getInstance() zal een Object teruggeven. Deze moet je casten naar de plugin nadat je het gekregen hebt van de container.
