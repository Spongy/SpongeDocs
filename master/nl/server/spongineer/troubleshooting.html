Solution: Get Java. Visit the Installing Java for more information.

Netwerk connectie fout (of DDOS aanval)Â¶

Symptomen: Netwerkverbinding is erg traag, valt constant uit of is totaal verdwenen.

Oplossingen: Check je internetverbinding naar de router of modem. Kijk of je browser dezelfde problemen ondervindt. Je kan een gratis service zoals speedtest.net gebruiken om je verbindingssnelheid te testen. Andere programma’s op je computer of lokale netwerk kunnen voor de problemen zorgen. Wees er ook zeker van dat je correct geportforward hebt naar je router en host-pc. Een DDoS-aanval, hoewel zeer onwaarschijnlijk, kan je hele internetverbinding platleggen. Je zult je ISP moeten contacteren voor extra hulp.

Niet genoeg vrij geheugenÂ¶

Symptomen: Server crasht, vaak met berichten over ‘Out of Memory’

Oplossing: Vergroot het maximum toegestane grootte van het geheugen met het opstart argument -XX:MaxPermSize=128. Vergroot je massa geheugen (als mogelijk) met de opstart argumenten: -Xms1024M (1GB opstart geheugen) en -Xmx2048M (2GB maximum). Hou in de gaten hoeveel vrij geheugen je op je computer overhoudt en kijk of er processen zijn die vastgelopen zijn. Waarschijnlijk moet je vast gelopen (Java) processen beÃ«indigen of je computer opnieuw opstarten. Een geheugenlek kan voorkomen als er een bug zit in een plugin, het kan tijd kosten voor dat de ontwikkelaar dat kan oplossen.

Nog steeds een probleem?: Als je nog steeds problemen ondervindt, hoewel je het bovenstaande hebt geprobeerd en niet de ‘heap size’ kunt verhogen, kijk dan in de ‘Task Manager’ om te zien of je alle werkgeheugen van je computer gebruikt. Als dit het geval is, is de enige optie om meer RAM toe te voegen aan je computer. Als er nog voldoende werkgeheugen over is, betekent dit dat je de 32-bits versie van Java draait. We raden je aan om over te stappen naar de 64-bits versie, als je besturingssysteem ook 64-bit ondersteunt.

Incorrecte configuratiebestanden (verkeerde formattering)Â¶

Symptomen: Ã‰Ã©n (of meerdere) plugins willen niet laden, of gedragen zich niet zoals ze zich zouden moeten gedragen. De server log bestanden zullen informatie bevatten over onleesbare bestanden bij het opstarten. De server kan hierdoor crashen en data kan corrupt raken.

Solution: Stop the server, and check your edited files. Load backup files of any corrupted data. You may need to delete a config file entirely and allow it to regenerate upon server startup.

Een plugin (of mod) werkt niet naar behorenÂ¶

Symptomen: Dit kan bijna alles zijn - plugins maken soms fouten, dus kijk of dit een heel specifiek probleem is met een of meerdere plugins. Meestal gaan dit soort fouten gepaard met foutmeldingen in de server logs.

Solution: Stop the server, and check to see nothing has been corrupted. Be sure to check that it isn’t from an incorrectly edited config file (above). Remove suspect plugins and add them again one by one, restarting the server each time. The problem may originate from one plugin that is out of date - check for updates. Plugin conflict may also be the cause, having two incompatible plugins.

Besturingssysteem is instabiel (door bijvoorbeeld virussen)Â¶

Symptomen: De server blijft herhaaldelijk crashen en andere delen van je besturingssysteem werken niet zoals ze houden moeten.

Oplossing: Stop alles. Bekijk nauwkeurig je systeem en opslag apparaten voor malware en virussen. Goede programma’s zijn AdwCleaner, Junkware Removal Tool, MalwareBytes, en de meeste antivirus software. Check je server bestand op beschadigingen nadat eventuele virussen of malware van je PC af is en dat je opnieuw hebt opgestart. Onderzoek of je hardware beschadigingen heeft als het probleem voor blijft doen.

Corrupte dataÂ¶

Symptomen: Bestanden waar de wereld bewaard wordt, is corrupt. Soms crasht de server of spelers als ze in een bepaald deel van de wereld komen. Kan onder andere te maken hebben met een corrupte database.

Oplossing: Laad de back-up bestanden voor de corrupte data. Er zijn programma’s die beschadigde werelden fixen en missende regio’s kunnen opnieuw gegenereerd worden. Ga opzoek naar de oorzaak van de corruptie, misschien was het wel een kapotte plugin, database driver, te weinig power of iets anders? Zorg ervoor dat je hoe dan ook altijd back-ups maakt van belangrijke data op een goed beveiligde plek.

Symptomen: Gister werkte alles nog prima, maar nu, nadat ik iets aangepast heb, werkt het niet meer..

Oplossing: SpongeDocs is niet groot genoeg om te omvatten wat mensen kunnen doen waardoor software onverwacht kapot gaat. Het werkt meestal het best om te bedenken welke recente aanpassingen je hebt gedaan, die je server heeft kunnen laten crashen of stroef loopt. Zit er bijvoorbeeld een geheugen kaart los na het afstoffen van je computer...

Symptomen: Geen enkele van de bovengenoemde symptomen en het werkt nog steeds niet zoals het zou moeten.

Solution: Time to get out the big guns. File a report on the SpongeForge or SpongeVanilla issue tracker, remembering to include details of the version of Forge and Sponge you are using, and a link to the relevant server log file.
