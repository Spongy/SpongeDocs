Sponge configureren¶

Alle configuratiebestanden zijn te vinden in de “config” map.

De config syntaxis¶

De meeste configuratie bestanden zullen gebruik maken van het HOCON formaat.

  • Introductie voor HOCON
  • JSON syntaxis

Wat jij kunt instellen¶

  • global.conf
  • server.properties

Plugins zullen ook hun eigen configuratiebestanden hebben in de “config” map.

Wereld configuratiebestanden¶

Er zijn drie typen wereld configuratiebestanden:

  • Globaal

  • Dimensie

  • Wereld

Globale configuratiebestanden kunnen effect hebben op alle werelden en dimensies. Dit is het standaardniveau van configuraties. Dimensie configuratiebestanden worden gebruikt om een bepaalde dimensie of groep werelden te beïnvloeden. Dit soort configuraties overschrijft de globale configuratiebestanden. Wereld configuratiebestanden worden gebruikt om individuele werelden aan te passen. Wereld configuraties overschrijven dimensie en globale configuraties.

De configuratie In-Game veranderen¶

Het is mogelijk om deze configuraties via een in-game opdracht, /sponge config, te wijzigen. De syntax voor de configuratie opdracht ziet er zo uit:

/sponge config <flag> <key> <value>

Er zijn flags om aan te geven welk doel u wilt veranderen. Deze flags zijn global, dimension en world.

  • -g is de vlag voor globaal

  • -d <dim> richt zich op een dimensie (vervang <dim> voor de dimensie die je wilt configureren)

  • -w <wereld>‘ richt zich op één wereld (vervang <wereld> met de door jou gekozen wereld).

key is de waarde die je wilt veranderen. value is hetgeen waarin je het wilt veranderen.

Hier is een voorbeeld van deze opdracht in actie:

/sponge config -d nether logging.chunk-load true

Dit zal de configuratie zeggen om te melden wanneer chunks geladen worden in the nether.

If you need to check the value of a key, you would need to omit the value. Checking the value of a key such as logging.chunk-load in the nether would be done like so:

/sponge config -d nether logging.chunk-load

Opslaan van een Wereld Configuratie¶

Saving a world config to the file may be desired after making modifications. This would be useful in the event of an unexpected server crash. This would be done by using the /sponge save command on the sponge server. The syntax for this command is similar to the config command:

/sponge save <flag>

Hier is een voorbeeld voor het opslaan van de globale configuratie:

/sponge save -g

Herladen van een Wereld Configuratie¶

Sometimes it may be desired that a world config is reloaded while the server is still running. This would be useful if you have made changes to the local config file and would like to reload it for use on the live server. This is made possible by the command /sponge reload. The syntax for the command is as follows:

/sponge reload <flag>

Hier is een voorbeeld voor het herladen van het the end wereld configuratie bestand:

/sponge reload -d the_end
