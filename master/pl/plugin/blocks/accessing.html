Wejście do Block-ów¶

Informacje podstawowe¶

Bloki są najczęściej zidentyfikowane i dostępne przez ich Location. Ten punkt lokacji określonych jako Extent (fragment). W większości przypadków World (świat) będzie służył jako zakres.

import org.spongepowered.api.Sponge;
import org.spongepowered.api.world.Location;
import org.spongepowered.api.world.World;

public Location<World> getBlockAt(String worldName, int posX, int posY, int posZ) {
    World world = Sponge.getServer().getWorld(worldName).get();
    Location<World> blockLoc = new Location<World>(world, posX, posY, posZ);
    return blockLoc;
}

Ostrzeżenie

Należy zauważyć, że powyższy przykład nie może sprawdzić czy istnieje w danym world. getWorld(worldName).get() nie powiedzie się, jeżeli nie jest to nazwa worldu który jest załadowany.

Z tym obiektem Location możesz otrzymać dalsze informacje na temat bloku. Poniższy kod sprawdza, czy wywoływany blok jest rodzaju transparent.

import org.spongepowered.api.block.BlockType;
import org.spongepowered.api.block.BlockTypes;

public boolean isBanner(Location<World> blockLoc) {
    BlockType type = blockLoc.getBlock().getType();
    return type.equals(BlockTypes.STANDING_BANNER)
            || type.equals(BlockTypes.WALL_BANNER);
}

Wskazówka

Funkcja == może być używana w miejscu equals() jeżeli istnieje tylko jedno BlockType dla każdego bloku, jednak zalecane jest używanie equals().

Manipulatory Danych w Block-ach¶

Dane Block-u przetrzymywane są jako DataManipulator, podobnych do innych części interfejsu API. Ten zbiór posiada informacje na temat składników trzymanych przez nasz Block, orientacje, określonych typów (kamień czy granit) i tak dalej. Sprawdzanie wartości tych manipulatorów jest łatwe, po prostu trzeba sprawdzić kierunek Block-u DirectionalData.

import org.spongepowered.api.data.key.Keys;
import org.spongepowered.api.data.manipulator.mutable.block.DirectionalData;

public boolean isFacingNorth(Location<World> blockLoc) {
    Optional<DirectionalData> optionalData = blockLoc.get(DirectionalData.class);
    if (!optionalData.isPresent()) {
        return false;
    }
    DirectionalData data = optionalData.get();
    if (data.get(Keys.DIRECTION).get().equals(Direction.NORTH)) {
        return true;
    }
    return false;
}

Po pierwsze musimy wiedzieć jaki sub-interfejs DataManipulator potrzebujemy. Te, które mają zastosowanie do bloków znajdują się w pakietach org.spongepowered.api.data.manipulator.mutable i org.spongepowered.api.data.manipulator.mutable.block. Następnie możemy po prostu przekazać klase do metody get(DataManipulator), która należy do Location i powinna zwrócić Optional. Następnie musimy sprawdzić, czy nasz DataManipulator faktycznie istnieje dla naszego Block sprawdzając ifPresent(). Jeśli istnieje, wtedy możemy z niego korzystać.

Więcej na temat DataManipulator można znaleźć w dokumentacji danych.

Wskazówka

Jeśli Block nie będzie wspierał specyfikacji DataManipulator, takich jak DirectionalData to nie trzeba używać isPresent(). Po prostu usuń opcjonalne DataManipulator i pobierz obowiązkowe dane by użyć .get() na końcu instrukcji. Zauważ, że spowoduje to NullPointerException, jeśli kiedykolwiek Block przestanie pobierać szczególne dane DataManipulator.

Statusy Block-ów¶

BlockState zawiera BlockType, wszelkie DataManipulator``y i właściwości, które są stosowane do Blocku, a wszelkie ``BlockTrait są w Block-u. Przechowuje wszelkie niezmienne wartości dla określonego Block-u. Jednym z zastosowań ImmutableDataManipulator, jak pokazano niżej:

import org.spongepowered.api.block.BlockState;
import org.spongepowered.api.data.manipulator.immutable.ImmutableWetData;

public void isWet(Location blockLoc) {
    BlockState sponge = blockLoc.getBlock();
    if (!sponge.getType().equals(BlockTypes.SPONGE)) {
        return false;
    }
    Optional<ImmutableWetData> wetness = sponge.get(ImmutableWetData.class);
    return wetness.isPresent();
}

Więcej informacji na temat zmiennych i niezmiennych DataManipulator-ów można znaleźć w dokumentacja danych.

Właściwości Block-u¶

Block-i mogą zawierać pewne właściwości. Właściwością jest wstępnie ustawiona wartość, którą definiuje logika gry. Na przykład, Block-i mogą zawierać wartości spadanio-odporne, które mogą być używane do określenia czy pracujesz z, czy bez faktycznego sprawdzenia typu Block-u, czy może być jeden na drugim. Na przykład jeśli chcemy uzyskać odporność na wybuch i sprawdzić czy to jest większe lub / niż równe jeden, wyglądałoby to tak:

import org.spongepowered.api.data.property.DoubleProperty;
import org.spongepowered.api.data.property.block.BlastResistanceProperty;

public boolean blastResistanceGreaterThanOne(Location<World> blockLoc) {
    Optional<BlastResistanceProperty> optional =
        blockLoc.getProperty(BlastResistanceProperty.class);

    if(optional.isPresent()) {
        BlastResistanceProperty resistance = optional.get();
        DoubleProperty one = DoubleProperty.greaterThanOrEqual(1);
        return one.matches(resistance);
    }
    return false;
}

To będzie odporność na wybuch naszego Block-u i porównując go do nowych DoubleProperty, jako BlastResistanceProperty dziedziczy DoubleProperty. Metoda zwróci jeśli opór wybuchu naszego Block-u jest większy niż jeden, wartości umieszczone w matches(). Jeśli chcemy sprawdzić, czy to było mniej niż dwa, możemy zastąpić go jako lessThan().

Jeśli porównaliśmy dwa istniejące wcześniej właściwości, to Operator zajmie naszą pierwszą wartość, tworzymy wartość double. Jeżeli Operator jest DELEGATE, czyli brak operatora, następnie Operator zajmie drugą wartość, jako jeden w matches(). Porównanie zwróci wartość false, jeśli oba są jako DELEGATE. Przykładowo porównując dwa PoweredProperty jako BooleanProperty można zobaczyć poniżej:

import org.spongepowered.api.data.property.block.PoweredProperty;

public boolean areBlocksPowered(Location<World> blockLoc, Location<World> blockLoc2) {
    Optional<PoweredProperty> optional = blockLoc.getProperty(PoweredProperty.class);
    Optional<PoweredProperty> optional2 = blockLoc2.getProperty(PoweredProperty.class);

    if(optional.isPresent() && optional2.isPresent()) {
        PoweredProperty property1 = optional2.get();
        PoweredProperty property2 = optional2.get();
        BooleanProperty booleanProperty = BooleanProperty.of(property1);
        BooleanProperty booleanProperty2 = BooleanProperty.of(true);

        if(booleanProperty2.matches(property1)) {
            return booleanProperty.matches(property2);
        }
    }
    return false;
}

Drugi if sprawdza czy kontrolnie czy jedna z wartości jest prawdą. Jeżeli jest prawdą, i obydwa są równe obie wartości muszą być prawdziwe. W związku z tym eliminując konieczność sprawdzania drugiej wartości, teraz wiemy, że obydwa Block-i są zasilane.

Lista właściwości dostępnych dla Block można znaleźć w pakiecie org.spongepowered.api.data.property.block.

Cechy Block-ów¶

Cecha Blocku jest pewną wartością aktualnego stanu Block-u. Block może lub nie zawierać cechy Blocku w zależności od typu. Dla przykładu łóżko po wywołaniu BooleanTrait odpowiada BED_OCCUPIED. Jako wartość logiczna może mieć tylko dwie wartości, prawda lub fałsz, cecha BED-OCCUPIED może być tylko prawda lub fałsz. Zaznaczenie tej wartości jest proste, wystarczy wywołać metodę getTraitValue() na BlockState. Poniżej przedstawiono przykład z łóżkiem:

import org.spongepowered.api.block.trait.BooleanTraits;

public boolean isBedOccupied(Location<World> blockLoc) {
    if(blockLoc.getBlock().getType().equals(BlockTypes.BED)) {
        return blockLoc.getBlock().getTraitValue(BooleanTraits.BED_OCCUPIED).get();
    }
    return false;
}

Ostrzeżenie

Jeśli to możliwe, zaleca się używać DataManipulator zamiast BlockTrait w miarę możliwości jak one są tylko po to, aby być oznaczonym jako rezerwa dla zgodności modyfikacji.
