Konsepter¶

Egenskaper¶

Alle blokker er av en basistype. Eksempler på basistyper er jord, trapper og blader. For å gjøre forskjell på disse basistypene har hver blokk et sett med forskjellige egenskaper, som kan motta et begrenset sett med verdier (for eksempel podzol*jord, *mursteins*trapper, *eike*blader). En blokk kan ha flere egenskaper (slik som *østvendte *mursteins*trapper).

Eksempler på blokkegenskaper

minecraft:dirt[snowy=false,variant=default]
minecraft:dirt[snowy=true,variant=default]
minecraft:dirt[snowy=false,variant=grassless]
minecraft:dirt[snowy=true,variant=grassless]
minecraft:planks[variant=oak]
minecraft:planks[variant=spruce]
minecraft:planks[variant=birch]
minecraft:redstone_wire[east=up,north=up,power=0,south=up,west=up]
minecraft:redstone_wire[east=side,north=up,power=0,south=up,west=up]
minecraft:redstone_wire[east=none,north=up,power=0,south=up,west=up]
minecraft:redstone_wire[east=up,north=side,power=0,south=up,west=up]

Noen egenskaper, er derimot ephemeral – de eksisterer kun når spillet kjører. Verdiene er ikke lagret i noen lagringsfil fordi verdiene dens oppdages automatisk. For eksempel, Redstone ledninger, uansett om de er aktivert eller ikke kan de bli oppdaget basert på omgivelsene (er det en spake som er aktivert?). I dette tilfellet er power, egenskapen av minecraft:redstone_wire som illustrert over, en kortvarig egenskap.

I skrivende stund lagrer Minecraft fortsatt blokkdata til et gammelt format med 12 bits fil for en basistype (4096 mulige basistyper) og 4 bits for “metadata” (16 mulige verdier per basistype). Egenskaper bindes ikke direkte sammen med metadata av bakoverkompatibilitetsgrunner: for eksempel har furnace-blokken to basistyper (i ferd med å smelte noe versus ikke i ferd med å smelte noe), og ingen av de to typene bruker metadata i det hele tatt. På en annen side bruker stokker sin metadata fullt ut, men fordi kombinasjonen av egenskaper overstiger 16 mulige verdier (tenk tretype og retning), må stokker fordeles på to basistyper.

I fremtiden vil det bare være ett 16-bittall (65536 mulige kombinasjoner av basistype og egenskaper). Blokker vil bli gitt en ID automatisk, og denne tildelingen vil bli lagret i verdenslagringsfilen. Dette er som vist under:

0 => minecraft:dirt[snowy=false,variant=default]
1 => minecraft:dirt[snowy=true,variant=default]
2 => minecraft:dirt[snowy=false,variant=grassless]
3 => minecraft:dirt[snowy=true,variant=grassless]
4 => minecraft:planks[variant=oak]
5 => minecraft:planks[variant=spruce]
etc.

Tile Entity/flisenhetsdata¶

Med 65536 mulige kombinasjoner er det ikke mulig å lagre mye informasjon som innholdet i ryggsekken, så det finnes en annen måte noen blokker har data: flisenheter

Tile entities themselves are Java objects (like a Chest class). Normally, Minecraft code would access data in a tile entity by getting its instance and then calling it methods or fields, like a regular object (world.getTileEntity(position).getInventory()). When tile entities need to be written to the save file, they are stored in the NBT format.

Tips

Noen ting, som malerier, er faktisk enheter.

flisenheter kan imidlertid også overstyre gjengivelse slik at de ikke ser ut som en vanlig blokk, selv om dette på generelt grunnlag er ineffektivt og medfører et fall i bildefrekvensen til klienter.
