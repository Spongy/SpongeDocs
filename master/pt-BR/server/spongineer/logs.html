Ficheiros de registo do SpongeForge¶

O SpongeForge escreve vários ficheiros de registo na pasta /logs, dentro da pasta principal do teu servidor. Desde o Forge 1521, estes são:

  1. fml-junk-earlystartup.log
  2. fml-server-latest.log
  3. latest.log

fml-server-latest.log¶

Nota

Só são mostradas algumas linhas de exemplo, aqui. Para leres o registo completo, segue o link: SpongeForge 1521 fml-server-latest.log

[main/INFO] [FML/]: Forge Mod Loader version 11.14.3.1521 for Minecraft 1.8 loading
[main/INFO] [FML/]: Java is Java HotSpot(TM) 64-Bit Server VM, version 1.8.0_51, running on Windows 8.1:amd64:6.3, installed at ##PATH_TO_JAVA_HERE##
[main/DEBUG] [FML/]: Java classpath at launch is forge.jar
[main/DEBUG] [FML/]: Java library path at launch is ##PATH_TO_JAVA_HERE##

O registo de exemplo indica que estamos a usar:

  • Forge 11.14.3.1521 (Versão 1521)

  • Java 8 64bit Update 51
  • Windows 8.1 x64

  • a pasta onde o Java está instalado (linha 4)

Aviso

O SpongeForge não corre no Java 6 (1.6.x) nem no Java 7 (1.7.x). Se encontrares um erro a dizer que usas uma versão do Java anterior à 8, por favor atualiza para o Java 1.8.x e tenta novamente!

[main/DEBUG] [FML/]: Examining for coremod candidacy spongeforge-1.8-1521-2.1-DEV-750.jar
[main/INFO] [FML/]: Loading tweaker org.spongepowered.asm.launch.MixinTweaker from spongeforge-1.8-1521-2.1-DEV-750.jar

This indicates that SpongeForge 750 was found and loaded by Forge. For further help regarding the SpongeForge naming scheme, have a look here: Installing SpongeForge.

Ler ficheiros de registo¶

If you’re unsure on how to read a common crashlog, you’ll find help here, but first we need a crashlog. For this short introduction we will just use an example crash from the Depuração page: Example crashlog of an outdated SpongeForge build.

WARNING: coremods are present:
SpongeCoremod (sponge-1.8-1499-2.1DEV-575.jar)
Contact their authors BEFORE contacting forge

A primeira coisa que vais ver é um Warning de que os coremods estão presentes. Nada com que te devas preocupar aqui, isso não é um erro, só um aviso de que deves contactar o suporte do Sponge, e não do Forge.

java.lang.NoClassDefFoundError: org/spongepowered/api/event/game/state/GameStartingServerEvent

Podes encontrar o erro algumas linhas abaixo. Neste caso, trata-se de uma NoClassDefFoundError. Se não sabes exatamente o que isso significa, vê a nossa página de Depuração. Se for um erro comum, estará listado lá. Se não for, podes sempre pedir ajuda no fórum. Certifica-te de que forneces o crashlog completo.

Felizmente, os detalhes do teu sistema estão incluídos na parte inferior do crashlog:

Minecraft Version: 1.8
Operating System: Windows 8.1 (amd64) version 6.3
Java Version: 1.8.0_51, Oracle Corporation
Java VM Version: Java HotSpot(TM) 64-Bit Server VM (mixed mode), Oracle Corporation
Memory: 515666256 bytes (491 MB) / 782761984 bytes (746 MB) up to 1847590912 bytes (1762 MB)
JVM Flags: 0 total;
IntCache: cache: 0, tcache: 0, allocated: 0, tallocated: 0
FML: MCP v9.10 FML v8.0.99.99 Minecraft Forge 11.14.3.1521 5 mods loaded, 5 mods active
States: 'U' = Unloaded 'L' = Loaded 'C' = Constructed 'H' = Pre-initialized 'I' = Initialized 'J' = Post-initialized 'A' = Available 'D' = Disabled 'E' = Errored
UC     mcp{9.05} [Minecraft Coder Pack] (minecraft.jar)
UC     FML{8.0.99.99} [Forge Mod Loader] (forge.jar)
UC     Forge{11.14.3.1521} [Minecraft Forge] (forge.jar)
UC     Sponge{1.8-1499-2.1DEV-575} [SpongeForge] (minecraft.jar)
U      Core{unknown} [Core Plugin] (Core.jar)
Loaded coremods (and transformers):
SpongeCoremod (sponge-1.8-1499-2.1DEV-575.jar)

Isto indica que

  • o Minecraft 1.8 com Forge 1521 estava a correr no

  • Java 8 Update 51 (versão de 64bits) e que

  • 2 mods adicionais estavam instalados

    • SpongeForge 1.8-1499-2.1DEV-575 (que é a build #575) e

    • Core

Nota

Nota que há outros três mods instalados (mcp, FML, Forge) que são requeridos em qualquer servidor Forge para o normal funcionamento deste.

Assim, podemos deduzir o seguinte:

  • talvez o plugin tenha crashado o servidor

  • a versão do SpongeForge não corresponde com a versão do Forge: é necessária a versão 1499 e nós temos a versão 1521 instalada

se queres saber como resolver isto, vê a checklist na página de Depuração.
