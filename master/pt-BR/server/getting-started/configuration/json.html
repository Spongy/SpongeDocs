SIntaxe JSON¶

Este é um exemplo de um ficheiro whitelist.json com a formatação correta (apesar de os UUID-s serem fictícios). Deves utilizar a mesma sintaxe.

[
  {
    "uuid": "01234567-89ab-cdef-0123-456789abcdef",
    "name": "Notch"
  },
  {
    "uuid": "a0b1c2d3-e4f5-0617-2839-4a5b6c7d8e9f",
    "name": "sk89q"
  }
]

Regras de Formatação¶

  • Parênteses retos ([]) abrem e fecham o ficheiro

  • Cada item no ficheiro fica entre parênteses curvos ({})

  • Cada key e o seu valor correspondente ficam na mesma linha

  • Se existir mais de um item, tanto os itens como os pares key/valor são separados por vírgulas

  • Todas as strings estão dentro de aspas

  • Os UUID-s têm 32 carateres, e são escritos de forma hexadecimal (0-9, a-f).

  • Os símbolos dos UUID-s são agrupados. Primeiro está um grupo de 8 carateres, depois três grupos de 4 e finalmente um de 12. Os grupos são separados por travessões (-)
