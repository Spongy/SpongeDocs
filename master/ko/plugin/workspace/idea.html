IntelliJ IDEA 설정하기¶

이 문서에서는 Sponge API로 플러그인 개발을 위한 IntelliJ IDEA 작업 공간을 구성하는 방법에 대해 설명합니다.

프로젝트 시작하기¶

Sponge API 플러그인를 개발할 때 Gradle 이나 Maven을 사용하는것이 좋습니다. Gradle과 Maven은 Sponge API의 종속성을 관리 할 때 크게 도움이 됩니다. 이 프로젝트에는 ``. jar``파일을 포함시키는 것이 좋습니다.

Gradle¶

프로젝트 생성하기:

  • **IntelliJ IDEA**를 실행 합니다.

  • ``Create New Project``를 클릭 합니다.

  • 팝업에서 ``Gradle``을 선택하고 ``Next``를 클릭합니다.

  • Group ID, Artifact ID 그리고 Version 을 입력합니다.

    • Group ID 는 조직 이름 혹은 비슷한 것 이어야 합니다.

    • Artifact ID 는 프로젝트 이름에 해당합니다.

    • **Version**은 자신이 알맞게 설정하시면 됨니다.

  • ``Next``를 두번 클릭하고 프로젝트 이름을 입력하고 ``Finish``를 클릭하세요.

Sponge API의 종속성을 가져올려면:

  • 탐색기에서 ``build. gradle``파일에 종속성을 추가합니다.

  • :doc:`dependencies page<dependencies>`에서 gradle파일의 예제를 참조하세요

  • IntelliJ의 창 오른쪽에 있는 **Gradle 탭**을 열어 새로고침 버튼을 누르세요.

  • Gradle설정이 완료되었습니다. 지금부터 당신의 플러그인을 코딩 하실 수 있습니다.

Maven¶

프로젝트 생성하기:

  • **IntelliJ IDEA**를 실행 합니다.

  • ``Create New Project``를 클릭 합니다.

  • 팝업에서 ``Maven``을 선택하고 ``Next``를 클릭합니다.

  • Group ID, Artifact ID 그리고 Version 을 입력합니다.

    • Group ID 는 조직 이름 혹은 비슷한 것 이어야 합니다.

    • Artifact ID 는 프로젝트 이름에 해당합니다.

    • **Version**은 자신이 알맞게 설정하시면 됨니다.

  • ``Next``를 클릭합니다.

  • 프로젝트 이름을 입력하고 ``Finsh``를 클릭합니다.

Sponge API의 종속성을 가져올려면:

  • 탐색기에서 ``pom.xml``를 여세요.

  • 편집기 탬에서 ``Code > Generate``를 선택합니다.

  • ``Dependency``를 팝업에서 클릭합니다.

  • :doc:`official Sponge API Maven artifact <dependencies>`에서 **Search by artifact**를 검색하고 ``Add``를 클릭합니다.

  • 메세지가 표시되면, Maven에서 종속성을 추가합니다.
