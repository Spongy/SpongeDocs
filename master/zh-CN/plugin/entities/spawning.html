生成一个实体¶

你需要四样数据以指定如何在世界上生成实体： Entity 、 Location 、 Extent 、和 EntityType 。获取这些东西其实相当简单，你需要关心的只有需要插件代码中指定的 Location 和你想要生成的 Entity 的类型。

例如，让我们试着去生成一个爬行者：

import org.spongepowered.api.entity.Entity;
import org.spongepowered.api.entity.EntityTypes;
import org.spongepowered.api.event.cause.Cause;
import org.spongepowered.api.event.cause.entity.spawn.EntitySpawnCause;
import org.spongepowered.api.event.cause.entity.spawn.SpawnTypes;
import org.spongepowered.api.world.Location;
import org.spongepowered.api.world.World;
import org.spongepowered.api.world.extent.Extent;

import java.util.Optional;

public void spawnEntity(Location<World> spawnLocation) {
    Extent extent = spawnLocation.getExtent();
    Optional<Entity> optional = extent
        .createEntity(EntityTypes.CREEPER, spawnLocation.getPosition());
    if (optional.isPresent()) {
        Entity creeper = optional.get();
        extent.spawnEntity(creeper,
            Cause.source(EntitySpawnCause.builder()
            .entity(creeper).type(SpawnTypes.PLUGIN).build()).build());
    }
}

这样我们就指定了 Location 对应的地域（Extent），用于生成我们想要生成的实体（译者注：可能的地域包括区块（Chunk）或者世界（World）等）。然后，我们使用这一地域创建了一个实体，不过需要注意的是它只是创建了一个实体，还没有在对应的地域内生成这一实体。我们需要指定 Entity 的类型，和其具体的 Location 。

EntityUniverse#createEntity(EntityType, Vector3d) 方法返回一个 Optional ，因为一个 Location 可能不适合生成一个 Entity 。然后我们从 Optional 中得到 Entity 实例，并使用 Extent 在世界中生成实体。我们还要为生成实体指定一个 Cause 。生成 Entity 的最佳选择是 EntitySpawnCause 。在此示例中，我们声明我们的实体是使用插件生成的，尽管我们可以使用任何原因来最贴切地声明为我们是如何生成的实体，比如刷怪笼或刷怪蛋。
