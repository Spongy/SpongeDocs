设置 Gradle¶

一般说来，一切必要的编译一个 Sponge 插件的工作都可以通过在你的项目中使用 Gradle 并简单地把 SpongeAPI 添加到依赖（Dependency）中来完成：

repositories {
    mavenCentral()
    maven {
        name = 'sponge'
        url = 'http://repo.spongepowered.org/maven'
    }
}

dependencies {
    compile 'org.spongepowered:spongeapi:5.0.0'
}

然而，为了进一步把 插件的元数据 集成进 Gradle，我们提供了用于 Sponge 插件的一个额外的 Gradle 插件 （被称为 SpongeGradle）以尽可能简化对于使用 Gradle 的项目的必要配置。

使用 SpongeGradle¶

注解

我们推荐使用 最新的 Gradle 版本 以使用 SpongeGradle。这一 Gradle 插件不能在很早的 Gradle 版本上正常工作。

使用 SpongeGradle 非常简单，它允许你尽可能减少用于 Sponge 插件上必要的 Gradle 配置。此外，它把 插件的元数据 集成进了 Gradle，比如自动添加小组、项目名、版本号以及在生成脚本中的内置插件，因此你只需要在一个文件里更新你的插件版本。

下面是一个可以用于大部分插件的一个简单模板。 请确保你已经把小组 ID 替换成了对应于你的小组。

plugins {
    id 'org.spongepowered.plugin' version '0.6'
}

group = 'com.example' // TODO
version = '1.0-SNAPSHOT'
description = 'An example plugin'

dependencies {
    compile 'org.spongepowered:spongeapi:5.0.0'
}

这几行用于处理你通常会手动完成的大部分设置：

  • 基本的 Gradle Java 设置

  • 请设置你的项目使用 Java 8 编译

  • 添加 Sponge 的 Maven 库（和中央库，Maven Central）

  • 设置你的项目的插件名称为 plugin ID ，并全部使用小写字母

  • 在 插件的元数据 中自动包含项目名称、说明和版本。

手动设置插件 ID¶

转为情况下，Gradle 插件将使用你的项目名称的小写字母形式配置你的插件 ID（plugin ID）。如果你想要使用一个自定义的 plugin ID 而仍然使用 插件的元数据 的集成，你可以这样手动修改它：

sponge {
    plugin {
        id = 'mypluginid'
    }
}

覆盖默认值¶

默认情况下，Gradle 插件将使用 插件的元数据 设置你在项目属性中定义的插件名称、插件版本、和说明（ plugin name ， plugin version 和 description ）。你也可以手动覆盖它们：

sponge {
    plugin {
        meta {
            name = 'My Plugin'
            version = '1.0.0'
            description = 'This is a plugin'
        }
    }
}

你也可以完全删除一个默认值：

sponge {
    plugin {
        meta {
            description = null
        }
    }
}
