给每个人¶

Sponge 是什么？¶

Sponge是一个多样化的全新Minecraft API，Sponge的使命是使用插件来多样化Minecraft的游戏体验。关于Sponge的更多详细内容请阅读： :doc: 简介 或者是Sponge的 :doc: 发展历程。

运行 Sponge 需要什么？¶

Sponge（以及 Minecraft）需要 Java Runtime Environment 来正确运行。很明显，你同时需要一台计算机来运行服务端，除此之外你不需要任何东西。在这里了解更多如何选择正确的 Java 版本：安装 Java

从哪里能得到 Sponge？¶

Sponge服务端尚未完工，但是已经有支持Sponge核心的开发版模组和插件放出了，点击下载：:doc:

哪些Sponge实现 是可用的？¶

The Sponge Project currently develops two implementations which both use the SpongeAPI:

  1. SpongeForge, a coremod for Minecraft Forge, which is an existing Minecraft modding framework famous for spurring the Minecraft modding scene. Forge lacks a cross-version API, and this is where Sponge steps in. Sponge allows server owners to deploy Sponge plugins with ease, making server management easier. (SpongeForge was formerly known as Sponge, until it was renamed to avoid confusion).

  2. **SpongeVanilla**【独立版】，独立的Sponge API，运行于原版Minecraft服务器端上。（SpongeVanilla原名：Granite）

从哪里能得到Sponge的插件？¶

On our official plugin repository, which is called Ore. Development has already started on GitHub. Temporarily, plugins are hosted on the SpongeForums.

What happened to Bukkit?¶

The Bukkit project halted further development of their API and server modification. Shortly thereafter, one of the contributors to Bukkit sent a DMCA takedown notice to stop further distribution of CraftBukkit. He was within his legal right. Downloads, as well as source code, for CraftBukkit and its derivatives (such as Spigot and Cauldron) are no longer publicly available. If you want to know the reasons why this affected Sponge development, have a look at our history page: Sponge的历史

Can I run Bukkit Plugins with this?¶

Sadly no, not natively. Sponge is using its own API (Sponge API), while Bukkit is using the Bukkit API. However there is the possibility that a third-party plugin for Sponge implements the Bukkit API on top of Sponge API.

至服主们¶

作为一名服主，切换到Sponge会给我的服务器带来什么影响？¶

对于现有的Forge服务器，你需要下载Sponge并将它放入mods文件夹。服务器可以像其他Forge服务器一样正常启动。

对于非Forge服务器，也许需要选择Sponge原版–不依赖与Forge的实现版本。现在Sponge文档已经有一些从Bukkit或者Canary向Sponge迁移的指南。值得一提的是，很多来自Bukkit社区的开发者需要将他们的精力放在Sponge上使他们的插件可以运行在Sponge的服务器上。

地图也可以迁移。但是相关的插件数据则取决于插件作者是否提供转换程序。有的插件作者也许不会提供。

另外，我们不会提供对Bukkit插件的支持。但是可能有其他人使得在Sponge上运行Bukkit插件成为可能。

切换到Sponge会影响我的玩家么？¶

切换到Sponge不会影响你的玩家。如果你(服主)正确地迁移了数据，那么玩家就可以向往常那样进入服务器。他们不需要安装Forge，除非你的服务器也安装了Mod。

给开发者¶

我能用Sponge做什么？¶

Sponge provides a Plugin API. This means that you can create new content and gamemodes on the go. Have a look at our plugin pages to get a quick-start: 插件开发

我不能用Sponge做什么？/Sponge有什么限制？¶

Sponge can’t be used to create new blocks, textures, mobs on the clientside or any other content which would need clientside modifications. The Sponge API wont support sending mods or plugins to the client for now due to security concerns. However you can make use of the ForgeAPI for clients and create Sponge plugins for the serverside. It is even possible to use Sponge on the client-side, but for several tasks mods are still required.

我是Bukkit的插件开发者! 我为什么不能用Bukkit的API？¶

BukkitAPI包含了使用GPL协议的代码。这也是导致Bukkit在九月份下线的原因。我们使用MIT协议，这样就可以避免像Bukkit那样的悲剧。虽然这样并不能使我们摆脱Mojang的控制，因为他们的代码是专有的。但是我们相信Mojang是支持Mod的并且会一如既往的支持下去。

我可以用插件接入服务端内部吗？¶

访问内部数据(CraftBukkit称其为”NMS”或”net.minecraft.server”)可以通过Forge来实现。Forge提供了大量的反混淆过的字段名称。但是，使用这种技巧增大了插件崩溃的机率，你需要自己注意。
