server.properties 文件¶

默认文件¶

这是一份 Minecraft 1.8.1 原版服务器的默认server.properties文件

#Minecraft server properties
#(File modification datestamp)
spawn-protection=16
max-tick-time=60000
generator-settings=
force-gamemode=false
allow-nether=true
gamemode=0
enable-query=false
player-idle-timeout=0
difficulty=1
spawn-monsters=true
op-permission-level=4
resource-pack-hash=
announce-player-achievements=true
pvp=true
snooper-enabled=true
level-type=DEFAULT
hardcore=false
enable-command-block=false
max-players=20
network-compression-threshold=256
max-world-size=29999984
server-port=25565
server-ip=
spawn-npcs=true
allow-flight=false
level-name=world
view-distance=10
resource-pack=
spawn-animals=true
white-list=false
generate-structures=true
online-mode=true
max-build-height=256
level-seed=
use-native-transport=true
motd=A Minecraft Server
enable-rcon=false

属性详解¶

感谢 Minecraft Wiki <http://minecraft.gamepedia.com> 的编辑者们对这些解释作出的贡献。

Boolean 属性只有两种合法的值：true 和 false 。 **Integer**属性必须是整数，同时有一定的范围限制。**String**类型可以包含任何符号。

键

类型

默认值

说明
allow-flight

布尔数值

 false

允许生存模式的玩家飞行，如果有Mod允许这么做的话

当allow-flight开启时，作弊可能会更常见，因为这会使得作弊更容易。创造模式不受影响。

false - 不允许飞行 (在空中滞留5秒以上的玩家会被踢出)。.

true - 允许飞行。
allow-nether

布尔数值

 true

允许玩家进入下界。

false - 下界传送门不工作。

true - 允许玩家通过传送门进入下界。
announce-player-achievements

布尔数值

 true

当某个玩家获得成就时发送广播。
difficulty

整数 (0-3)

 1

定义服务器的难度。(比如怪物伤害，饥饿的处理和药水对玩家的效果)

0 - 和平

1 - 简单

2 - 普通

3 - 困难
enable-query

布尔数值

 false

允许GameSpy4协议，用于收集服务器信息。
enable-rcon

布尔数值

 false

允许远程连接到游戏控制台。
enable-command-block

布尔数值

 false

允许命令方块。
force-gamemode

布尔数值

 false

强制玩家进入时在默认的游戏模式。

false - 玩家加入时的游戏模式是上次他们退出时的游戏模式。

true - 玩家总是以默认游戏模式加入游戏。
gamemode

整数 (0-3)

 0

定义游戏模式。

0 - 生存

1 - 创造

2 - 冒险

3 - 旁观
generate-structures

布尔数值

 true

是否在新区块中生成结构。 (例如村庄)

false - 不会生成结构。

true - 会生成结构。

注意： 即使设置为false，地牢依然会生成。
generator-settings string blank

这个设置用于自定义世界生成。访问 Minecraft Wiki 的 Superflat 和 Customized 页面(外部链接)以获得设置样例。
hardcore

布尔数值

 false

如果设置为 true，玩家在死后会被永久封禁。
level-name string world

“level-name”选项会被用于世界的名称和文件夹的名称。你可以将已有的世界目录拷贝过来，并修改这里和目录名一致以载入世界。

单引号(‘)等符号可能需要在之前使用反斜线(\)转义。
level-seed string blank

添加一个地图种子，在单机模式中。

一些例子，如：minecraft,404,1a2b3c
level-type string

默认

确定地图的类型。

DEFAULT - 标准世界的丘陵、 山谷、 水等。

FLAT - 一个平坦的世界，没有特色，用于建筑。

LARGEBIOMES - 与默认类似，但所有生物群落相同较大。

AMPLIFIED - 与默认类似，但世界生成高度限制增加。

CUSTOMIZED - 与默认一样，除非generator-settings被设置为一个预设。
max-build-height

整数

 256

允许建造方块的最高高度, 地形也会在高度限制内十分自然的生成.
max-players

整数 (0-2147483647)

 20

在同一服务器上可以同时容纳玩家的最大数目. 越多玩家同时在线需要更多的服务器资源, 请注意: 有OP权限的玩家将不被计算在线玩家数目中. 但OP不能在服务器满人的情况下加入游戏. 如果你把这个值写的特别大, 游戏会爆炸(逃
max-tick-time

整数 (0-(2 ^63-1))

 60000

单个Tick的最长等待时间, 如果服务器发现超过这个值, 将会输出以下消息: “A single server tick took 60.00 seconds (should be max 0.05); Considering it to be crashed, server will forcibly shutdown”. 接着就会调用System.exit(1)来结束服务器运行.

-1 - 禁用此功能
max-world-size

整数 (1-29999984)

 29999984

This sets the maximum possible size in blocks, expressed as a radius, that the world border can obtain. Setting the world border bigger causes the commands to complete successfully, but the actual border will not move past this block limit. Setting the max-world-size higher than the default doesn’t appear to do anything.

示例

  • 设定世界大小为1000会让你的世界的长宽为2000x2000

  • 设定世界大小为4000会让你的世界的长宽为8000x8000
motd string A Minecraft Server

在服务器名字下的那一行小字(服务器描述).

  • 公告支持颜色代码和字体代码。

  • 如果MOTD有超过59个字符, 客户端显示MOTD时可能会出现一个通讯错误.
network-compression-threshold

整数

 256

By default it allows packets that are n-1 bytes big to go normally, but a packet that n bytes or more will be compressed down. So, lower number means more compression but compressing small amounts of bytes might actually end up with a larger result than what went in.

-1 - disable compression entirely

0 - compress everything

注意： 以太网规范 要求小于64字节的数据包填充为64字节。 因此，将这个值设置为小于64可能带来坏处。 同时也不建议超过 *MTU（最大传输单位） * ，通常为1500字节
online-mode

布尔数值

 true

服务器会检查MOJANG服务器内是否有这个玩家. 如果你的服务器没有连接到互联网上. 请把它设置为false. 如果服务器对盗版玩家开放, 也请把他设置为false. 那么服务器就会在离线模式下运行. 如果你启用了正版验证, 且验证服务器出现问题, 任何人都无法加入服务器.

false - 已禁用。服务器不会检查玩家是否为正版.

true - 服务器将会检查玩家是否为正版.
op-permission-level

整数 (1-4)

 4

设置OP的权限等级。每个等级也包含比其低等级的权限

1 - Op可以绕过出生点保护.

2 - Op可以使用 /clear, /difficulty, /effect, /gamemode, /gamerule, /give, 和 /tp, 并且可以编辑命令方块.

3 - Op可以使用 /ban, /deop, /kick, 和 /op.

4 - Op可以使用 /stop.
player-idle-timeout

整数

 0

值不为0时, 该功能启用, 玩家在这个闲置时间后被自动踢出.

** 请注意： ** 有时服务器空闲时间将会被重置，例如，当服务器接收到以下数据包时：

  • 102 (0x66) WindowClick
  • 108 (0x6c) ButtonClick
  • 130 (0x82) UpdateSign
  • 14 (0xe) BlockDig
  • 15 (0xf) Place
  • 16 (0x10) BlockItemSwitch
  • 18 (0x12) ArmAnimation
  • 19 (0x13) EntityAction
  • 205 (0xcd) ClientCommand
  • 3 (0x3) Chat
  • 7 (0x7) UseEntity
pvp

布尔数值

 true

启用服务器的PVP, 玩家们就可以互相伤害了.

false - 玩家们不能互相伤害 (也就是PvE模式 (PvE)).

true - 来啊 互相伤害啊.

注意: 由别的玩家制造的非直接伤害 (*被别人的岩浆烧/TNT爆炸/水淹/沙子盖住) 在PVP关闭的情况下还是有伤害的.*
query.port

整数 (1-65534)

 25565

设置查询服务器的端口(需要开启这个功能**enable-query**)
rcon.password string blank

设置远程连接的密码。
rcon.port

整数 (1-65534)

 25575

设置RCON远程连接的端口.
resource-pack string blank

设置资源包的路径. 玩家可以选择是否使用服务器提供的资源包.
resource-pack-hash string blank

资源包的SHA-1值，请填写为小写十六进制。 建议指定这个值。 这不用于验证资源包的完整性，但是提高了缓存的有效性和可靠性。
server-ip string blank

为服务器设置监听的地址. 强烈建议将这个选项留空.
server-port

整数 (1-65534)

 25565

设置服务器要监听的端口. 如果你的服务器没有公网IP/在防火墙之后, 请记得添加端口转发.
snooper-enabled

布尔数值

 true

设置是否服务器定期向 http://snoop.minecraft.net 发送 snoop 数据。（外部链接）

false - 不发送数据。

true - 发送数据。
spawn-animals

布尔数值

 true

设定敌对生物是否生成

false - 所有动物都会消失, 并且不再生成.

true - 动物跟往常一样正常生成

提示：如果你觉得卡顿和延迟，请设置为false
spawn-monsters

布尔数值

 true

设置敌对生物是否能够生成

false - 所有怪物都会立即消失, 也不再会生成.

true - 怪物跟往常一样生成: 在黑暗中或是在晚上生成.

如果你设置难度为和平模式. 该选项无效.

提示：如果你觉得卡顿和延迟，请设置为false
spawn-npcs

布尔数值

 true

确定村民是否生成

true - 启用. 将会生成村民.

false - 禁用. 没有村民
spawn-protection

整数

 16

决定出生点保护半径. 0: 关闭出生点保护. 1: 保护出生点附近的3x3范围 2: 保护出生点附近的5x5范围 X: 以此类推 在第一个玩家还没进入服务器之前, 出生点保护还不会启用. 如果你的服务器没有任何一个OP, 也不会启用.
use-native-transport

布尔数值

 true

Linux 服务器性能改进: 在 Linux 上优化数据包发送/接收。

false - 已禁用. 禁用 Linux 数据包收发优化。

true - 已启用. 启用 Linux 数据包收发优化。
view-distance

整数

(3-15)

 10

设置一个服务端世界数据发送给客户端，以玩家半径为准，它将决定服务端观看距离。

10 是默认值. 如果你感觉服务器卡顿, 请减小这个值.
white-list

布尔数值

 false

开启服务器白名单. See 管理白名单. 白名单开启时不在白名单内的玩家不能进入服务器. 例如你可以用于自己的私人服务器, 如果你只想让你钦定的朋友进入, 请把他们加入到白名单内.

false - 不使用白名单。

true - 文件 whitelist.json 用来生成玩家白名单.

注意: 游戏内钦定的OP会被自动加入到这个列表里 你不需要再次把他们加入进来.
