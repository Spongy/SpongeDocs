SpongeForge Logdateien¶

SpongeForge speichert mehrere logfiles im logs Ordner in deinem Serverordner. In Forge Build 1521 sind dies:

  1. fml-junk-earlystartup.log
  2. fml-server-latest.log
  3. latest.log

fml-junk-earlystartup.log¶

Bemerkung

Du siehst hier nur ein paar Beispielzeilen. Klicke hier um das vollständige Beispiel anzusehen: SpongeForge 1521 fml-junk-earlystartup.log file

fml-server-latest.log¶

Bemerkung

Du siehst hier nur ein paar Beispielzeilen. Klicke hier um das vollständige Beispiel anzusehen: SpongeForge 1521 fml-server-latest.log

[main/INFO] [FML/]: Forge Mod Loader version 11.14.3.1521 for Minecraft 1.8 loading
[main/INFO] [FML/]: Java is Java HotSpot(TM) 64-Bit Server VM, version 1.8.0_51, running on Windows 8.1:amd64:6.3, installed at ##PATH_TO_JAVA_HERE##
[main/DEBUG] [FML/]: Java classpath at launch is forge.jar
[main/DEBUG] [FML/]: Java library path at launch is ##PATH_TO_JAVA_HERE##

Das Beispiel zeigt das wir folgendes Setup haben:

  • Forge 11.14.3.1521 (Version 1521)
  • Java 8 64bit Update 51
  • Windows 8.1 x64

  • the Verzeichnis in das Java installiert wurde (Zeile 4)

Warnung

SpongeForge läuft nicht auf Java 6 (1.6.x) oder Java 7 (1.7.x). Wenn du einen Fehler siehst, der besagt das du eine ältere Java-Version als Java 8 nutzt, aktualisiere bitte zuerst deine JRE auf Version 1.8.x und versuche dann erneut den Server zu starten!

[main/DEBUG] [FML/]: Examining for coremod candidacy spongeforge-1.8-1521-2.1-DEV-750.jar
[main/INFO] [FML/]: Loading tweaker org.spongepowered.asm.launch.MixinTweaker from spongeforge-1.8-1521-2.1-DEV-750.jar

Dies bedeutet das SpongeForge 750 gefunden und von Forge geladen wurde. Wenn du mehr über die Benennung von SpongeForge wissen möchtest, schau hier nach: SpongeForge installieren

Logdateien lesen¶

Wenn du dir nicht sicher bist, wie man einen Crashlog liest, wirst du hier Hilfe finden. Für diese kurze Einführung werden wir ein Beispiel von der debuggin Seite einsetzen: Example crashlog of an outdated SpongeForge build

WARNING: coremods are present:
SpongeCoremod (sponge-1.8-1499-2.1DEV-575.jar)
Contact their authors BEFORE contacting forge

Das Erste was dir auffällt ist die Warning das Coremods installiert sind. Mach dir keine Sorgen, da dies kein Fehler sondern nur ein Hinweis ist.

java.lang.NoClassDefFoundError: org/spongepowered/api/event/game/state/GameStartingServerEvent

Ein paar Zeilen darunter findest du den eigentlichen Fehler. In diesem Fall ist es ein NoClassDefFoundError. Wenn du dir unsicher bist was das bedeutet, schau auf der Debuggen Seite nach, wenn es ein üblicher Fehler ist, ist er dort gelistet. Ist es keiner, kannst du immernoch in unseren Foren um Hilfe bitten. Achte darauf das vollständige Crashlog anzuhängen.

Glücklicherweise sind die Systemdetails am Ende des Crashlogs mit angehangen:

Minecraft Version: 1.8
Operating System: Windows 8.1 (amd64) version 6.3
Java Version: 1.8.0_51, Oracle Corporation
Java VM Version: Java HotSpot(TM) 64-Bit Server VM (mixed mode), Oracle Corporation
Memory: 515666256 bytes (491 MB) / 782761984 bytes (746 MB) up to 1847590912 bytes (1762 MB)
JVM Flags: 0 total;
IntCache: cache: 0, tcache: 0, allocated: 0, tallocated: 0
FML: MCP v9.10 FML v8.0.99.99 Minecraft Forge 11.14.3.1521 5 mods loaded, 5 mods active
States: 'U' = Unloaded 'L' = Loaded 'C' = Constructed 'H' = Pre-initialized 'I' = Initialized 'J' = Post-initialized 'A' = Available 'D' = Disabled 'E' = Errored
UC     mcp{9.05} [Minecraft Coder Pack] (minecraft.jar)
UC     FML{8.0.99.99} [Forge Mod Loader] (forge.jar)
UC     Forge{11.14.3.1521} [Minecraft Forge] (forge.jar)
UC     Sponge{1.8-1499-2.1DEV-575} [SpongeForge] (minecraft.jar)
U      Core{unknown} [Core Plugin] (Core.jar)
Loaded coremods (and transformers):
SpongeCoremod (sponge-1.8-1499-2.1DEV-575.jar)

Dies zeigt, dass

  • Minecraft 1.8 mit Forge 1521 auf

  • Java 8 Update 51 (64-Bit-Version) genutzt wird und das

  • 2 zusätzliche Mods installiert sind

    • SpongeForge 1.8-1499-2.1DEV-575 (das ist Version #575) und

    • Core

Bemerkung

Bitte beachte, dass die anderen drei Mods (mcp, FML, Forge) auf jedem Forge Server installiert sein müssen damit der Server ordnungsgemäß funktioniert.

Jetzt können die folgenden Annahmen getroffen werden:

  • vielleicht lies das Plugin den Server abstürzen

  • SpongeForge passt nicht zur Forge-Version: 1499 benötigt, 1521 installiert

Wenn du wissen willst, wie man diese Probleme behebt, schau dir unsere Checkliste auf der Debuggen Seite an.
