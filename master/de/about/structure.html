The Structure of the Sponge Project¶

The Sponge Project consists of different subprojects, hosted in various repositories on GitHub. Here’s a short overview before going into detail:

Project Description What is done in the repository?
SpongeAPI The API itself Development of the API itself
SpongeForge A SpongeAPI implementation built on top of Forge Development of the parts of SpongeForge which rely on Forge
SpongeVanilla A SpongeAPI implementation built directly on top of Vanilla Minecraft Development of the Vanilla Counterpart of the SpongeForge repository
SpongeCommon The shared code between SpongeForge and SpongeVanilla Development of all code which is shared between SpongeForge and SpongeVanilla
Mixin

Das Werkzeug das wir nutzen um unsere Implementierungen mit der darunter liegenden Struktur zu verknüpfen

 Development of our solution to hook Sponge into the Minecraft server
SpongeDocs The official SpongeProject Documentation Expanding, fixing and writing the SpongeDocs
Ore Plugin hosting solution Development of our plugin hosting solution

SpongeCommon, SpongeForge and SpongeVanilla¶

SpongeCommon ist die Basis unserer Implementierungen. Es enthält alles Code, der von SpongeForge und SpongeVanilla geteilt wird. Das SpongeForge und SpongeVanilla Repository dagegen enthalten jeweils den Code, der nicht geteilt werden kann, da Forge einige spezielle Anpassungen erfordert, die nicht auf Vanilla Minecraft funktionieren und umgekehrt.

Wenn man auf SpongeForge (die Implementierung) verweist, so ist immer alles gemeint, das in den SpongeCommon und SpongeForge Repositories enthalten ist. Das Gleiche trifft auch auch SpongeVanilla und SpongeCommon zu. Das ist der Grund weshalb das Komplilieren von SpongeForge oder SpongeVanilla ohne SpongeCommen mit einzubeziehen nicht funktioniert.

Ore¶

Ore is our very own plugin hosting solution. It’s written in Scala, based on the Play Framework and of course open-source!
